resolver.resolve is a linear firstOrNull over all 51 states per row (bbox check, then ray-cast) = O(rows x states). At 200k it is ~3s, fine. As the dataset grows this loop is the seed bottleneck; a coarse spatial index over the state bounding boxes (grid bucket, or sort-by-bbox / STRtree) would shrink the per-point candidate set. Worth noting given the brief cares about growth, though it stays a one-time cost.

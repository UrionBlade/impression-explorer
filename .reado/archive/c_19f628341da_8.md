Empty-polygon safety: if a StateShape is built with an empty polygon list (SeedRunner returns emptyList for a non-Polygon/MultiPolygon geometry), BBox.of leaves min=+MAX and max=-MAX, so contains() tests an empty range (a..b with a>b) and always returns false - the state silently matches nothing. Correct and crash-free, but it hinges on Kotlin empty-range semantics; a later change to the init or any clamping could flip it into matching everything. An assertion or a comment pinning the intent would harden it.

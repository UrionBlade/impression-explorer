The missing-key fallback (?? catalogs.en[key] ?? key) works at runtime (JS returns undefined for an absent key), but the type Record<Locale, Record<string,string>> makes catalogs[locale][key] a plain string, so TS sees the fallbacks as unreachable and will not warn about a missing key. The vitest catalog-parity test is the real guard here. Enabling noUncheckedIndexedAccess would make the type reflect runtime (string | undefined) and justify the fallback.

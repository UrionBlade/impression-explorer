GSAP owns the states opacity here, but React also declares it: the base paths render with style opacity 0 (line 118) while this effect animates them to 1 on the DOM. It works today only because React skips rewriting a style value it considers unchanged (0 -> 0), so GSAPs 1 survives later re-renders (data load, theme toggle, hover). That is fragile: any change that makes React re-assert opacity would flash the whole map back to invisible. Cleaner to let GSAP fully own it — set the initial 0 via gsap.set in this effect (or animate a wrapping <g>) and drop opacity from the paths React style.

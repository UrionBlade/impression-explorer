# Impeccable — Project Design Context

_Captured for the Impression Explorer challenge. Guides all UI/UX work. Update in place as decisions evolve._

## Design Context

### Users
Data/marketing **analysts** at (or evaluating) Cuebiq. They explore where advertising impressions concentrate across the US and answer specific questions (per device, per hour, per state, Black Friday over the years). Context of use: focused desktop analysis, comparing and drilling into aggregates. They value **trust in the numbers** and **speed of insight** — but this is also a hiring showcase, so the experience must feel like a **polished product**, not a dashboard demo.

### Brand Personality
Three words: **precise, refined, alive.** Voice is calm and expert — never loud or gimmicky. The interface should evoke **confidence** (the data is credible) and **quiet delight** (it's a pleasure to use). Bilingual **Italian / English**, English default.

### Aesthetic Direction
**Elegant / editorial.** Generous typographic hierarchy, considered whitespace, a "report you'd want to read" feel — closer to a beautifully typeset data story than a control panel. **Medium information density**: not one chart per screen, not everything crammed together — grouped, breathing panels an analyst can scan.

- **Theme**: both light and dark, **dark as default** (the map and density scales read best on a dark ground).
- **Palette**: **derived from the official Cuebiq logo** (navy wordmark + teal→blue diamond mark). Core brand hex (sampled from the logo, fine-tune against an official brand kit if provided):
  - Ink / navy (brand): `#232842` — dark-theme text, light-theme headings, outlines
  - Cuebiq blue (royal, primary accent): `#3D6DB5`
  - Cuebiq light blue: `#7FA8D9`
  - Cuebiq teal (secondary accent): `#2AC3C4`
  - Dark surfaces (tinted toward navy, not pure black): `#11152A` bg, `#1A2140` panels
  - Light theme: `#F7F9FC` bg, `#232842` ink text
  - **Choropleth / density sequential scale**: the brand mark's own gradient — teal → blue → navy (`#2AC3C4 → #3D6DB5 → #232842`) for low → high density; reads as "more saturated/darker = more impressions" and is on-brand. Direction/lightness adapted per theme.
- **Anti-references**: generic Bootstrap admin templates; neon/gamer gradients; cluttered BI tools (Power BI defaults). Not playful/cartoonish, not corporate-sterile.

### Motion (first-class)
This product is **animated throughout** — chart reveals, transitions, and user interactions — in service of the "cool product" goal, but kept **editorial and purposeful**, never decorative noise. Motion stack, with clear division of labor to avoid overlap:
- **Lenis** — smooth scrolling / scroll feel across the app.
- **GSAP (+ ScrollTrigger)** — scroll-driven sequences and **data/chart animations** (values counting up, bars/areas drawing in, choropleth states filling).
- **Framer Motion** — React component & layout transitions, page/route changes, micro-interactions (hover, press, enter/exit).
- Every animation MUST respect `prefers-reduced-motion`: reduced/removed motion path is required, not optional.

### Design Principles
1. **Data first, ornament never** — motion and style amplify comprehension (draw the eye to the change), they don't compete with it.
2. **Editorial calm** — strong type hierarchy and whitespace over dense chrome; let results breathe.
3. **Purposeful motion** — every animation has a reason (reveal, transition, feedback); default to subtle, honor reduced-motion.
4. **Legible in both themes and both languages** — contrast (WCAG AA), number/date formatting, and layout hold up in light/dark and IT/EN.
5. **Scales gracefully** — the UI stays fast and calm as the dataset grows (aggregate server-side, animate the summary, never choke on raw volume).

---

_Accessibility baseline: WCAG 2.1 AA contrast, full keyboard operability, visible focus, `prefers-reduced-motion` honored, color never the sole signal (pair choropleth/scale color with labels/values)._
